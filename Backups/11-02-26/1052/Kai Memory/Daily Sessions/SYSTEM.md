## How Kai Remembers Things

### 1. MEMORY.md (Read Every Session)
**Contains:** Identity, core principles, critical patterns, key decisions
**Size:** Keep it concise (can grow, but I read it all)
**Update:** When something fundamental changes

### 2. trading/rules.md (Read When Trading)
**Contains:** Your specific entry/exit criteria, risk rules, routine
**Size:** Comprehensive but structured
**Update:** When your model evolves

### 3. trading/lessons.md (Review Weekly)
**Contains:** Mistakes, insights, wins, pattern recognition
**Size:** Grows over time
**Update:** After every trade/session

### 4. memory/YYYY-MM-DD.md (Daily Logs)
**Contains:** Raw session notes, what happened today
**Size:** Daily, archived
**Update:** Every session automatically
**Cleanup:** Weekly review, distill into lessons/memory

### 5. trading/model-research.md (Reference)
**Contains:** Intel on TTrades/GxT models, links, notes
**Size:** Comprehensive reference
**Update:** As I learn more

## Maintenance Schedule

**Every Session:**
- Read MEMORY.md
- Read USER.md
- Read SOUL.md
- Read today's memory file
- Read trading/rules.md (if trading)

**Weekly (Heartbeat):**
- Review week's daily notes
- Update lessons.md with insights
- Distill into MEMORY.md if needed
- Archive old daily files

**Monthly:**
- Review all lessons
- Update rules based on performance
- Clean up old files

---

## Token Efficiency Tips

**Do:**
- Summarize long conversations before ending session
- Put structured data in files (tables, lists)
- Reference files instead of repeating content

**Don't:**
- Log every tiny detail to MEMORY.md
- Keep redundant information across files
- Let daily notes grow unbounded
